Transition

Add effects to an element when it is transformed
from one style to another without using Flash animation or JavaScript.

transition

  • Specify which CSS property you want to add the effect to.
    eg. Specifies when the mouse pointer is over the
    element:
        .transition:hover { width:300px; }
  • Specify duration of the specified effect.
    ** If the duration is not specified, there will be no transition effect because the default value is 0.

Multiple changes

To add transition effects to multiple styles, add multiple attributes separated by commas.

Animation

Animations can replace animated images, Flash animations, and JavaScript on many web pages.

Some older browsers need specific prefixes (-webkit-). eg. -webkit-animation-name; @-webkit-keyframes;

@keyframes

Gradually change from the current style to the new style at certain times.
Must bind the animation to an element. eg. bind ABC to

element: 
          @keyframes ABC {
            from {background-color: red;}
            to {background-color: yellow;}
          }
          
          div {
            width: 100px;
            height: 100px;
            background-color: red;
            animation-name: ABC;
            animation-duration: 4s;
          }

animation-name

Bind keyframe name to the element.

animation-duration

If animation-duration property is not specified, no animation will occur, because the default value is 0s.

By using percent, you can add as many style changes as you like. 

            @keyframes BCD {
              0%   {background-color: red;}
              25%  {background-color: yellow;}
              50%  {background-color: blue;}
              100% {background-color: green;}
            }
            
            div {
              width: 100px;
              height: 100px;
              background-color: red;
              animation-name: BCD;
              animation-duration: 4s;
            }

animation-delay

Specifies a delay for the start of an animation. 

            div {
              width: 100px;
              height: 100px;
              position: relative;
              background-color: red;
              animation-name: example;
              animation-duration: 4s;
              animation-delay: 2s;
            }

animation-iteration-count

Specifies the number of times an animation should run. 

            div {
              width: 100px;
              height: 100px;
              position: relative;
              background-color: red;
              animation-name: example;
              animation-duration: 4s;
              animation-iteration-count: infinite;
            }

animation-direction

  • normal - The animation is played as normal (forwards). This is default.
  • reverse - The animation is played in reverse direction (backwards).
  • alternate - The animation is played forwards first, then backwards.
  • alternate-reverse - The animation is played backwards first, then forwards
            div {
              width: 100px;
              height: 100px;
              position: relative;
              background-color: red;
              animation-name: example;
              animation-duration: 4s;
              animation-iteration-count: 2;
              animation-direction: alternate;
            }

animation-timing-function

Specifies the speed curve of the animation.

  • linear - Specifies an animation with the same speed from start to end.
  • ease - Specifies an animation with a slow start, then fast, then end slowly (this is default).
  • ease-in - Specifies an animation with a slow start.
  • ease-out - Specifies an animation with a slow end.
  • ease-in-out - Specifies an animation with a slow start and end.
  • cubic-bezier(n,n,n,n) - Lets you define your own values in a cubic-bezier function.

animation-fill-mode

Specifies a style for the target element when the animation is not playing (before it starts, after it ends, or both).

  • none - Default value. Animation will not apply any styles to the element before or after it is executing.
  • forwards - The element will retain the style values that is set by the last keyframe (depends on animation-direction and animation-iteration-count).
  • backwards - The element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during the animation-delay period.
  • both - The animation will follow the rules for both forwards and backwards, extending the animation properties in both directions.

animation

            div {
              animation-name: example;
              animation-duration: 5s;
              animation-timing-function: linear;
              animation-delay: 2s;
              animation-iteration-count: infinite;
              animation-direction: alternate;
            }
This is equal to 
            div {
              animation: example 5s linear 2s infinite alternate;
            }
