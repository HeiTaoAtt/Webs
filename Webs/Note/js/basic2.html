Regular Expression

a sequence of characters that forms a search pattern. /pattern/modifiers;
//* can be a single character, or a more complicated pattern.
//* can be used to perform all types of text search and text replace operations.

var patt = /w3schools/i; 
// "/w3schools/i"  is a regular expression. 
// "w3schools"  is a pattern (to be used in a search).
// "i"  is a modifier (modifies the search to be case-insensitive).

  1. String Methods:

    • search(): searches a string for a specified value and returns the position of the match.
      using string search() with a string:
      var str = "Visit W3Schools!";
var n = str.search("W3Schools");
      using string search() with a regular expression:
      var str = "Visit W3Schools";
var n = str.search(/w3schools/i);
// returns 6
    • replace(): replaces a specified value with another value in a string.
      using replace() with a string:
      var str = "Visit Microsoft!";
var res = str.replace("Microsoft", "W3Schools");
      using replace() with a regular expression:
      var str = "Visit Microsoft!";
var res = str.replace(/microsoft/i, "W3Schools");

  2. Regular expression Modifiers: perform searches.

    modifier description
    i Perform case-insensitive matching
    g Perform global match (find all matchs)
    m Perform multiline matching 
    function myFunction() {
var str = "Is this all there is?";
var result1 = str.match(/is/i);
var result2 = str.match(/is/g);
var result3 = str.match(/is/m);
document.getElementById("demo").innerHTML = result;
}
// return Is
// return is,is
// return is

    function myFunction() {
var str = "\nIs th\nis it?";
var result1 = str.match(/is/i);
var result2 = str.match(/is/g);
var result3 = str.match(/is/m);
document.getElementById("demo").innerHTML = result;
}
// return Is
// return is
// return is

  3. Regular expression Patterns: find a range of characters.

    • Brackets:
      bracket description
      [abc] Find any of the characters between the brackets
      [0-9] Find any of the digits between the brackets
      (x|y) Find any of the alternatives separated with |
    • Metacharacters: characters with a special meaning.
      metacharacter description
      \d Find a digit
      \s Find a whitespace character
      \b Find a match at the beginning of a word like this: \bWORD, or at the end of a word like this: WORD\b
      \uxxxx Find the Unicode character specified by the hexadecimal number xxxx
    • Quantifiers: define quantities.
      quantifier description
      n+ Matches any string that contains at least one n
      n* Matches any string that contains zero or more occurrences of n
      n? Matches any string that contains zero or one occurrences of n

  4. Regular expression Object:
    -test(): searches a string for a pattern, and returns true or false, depending on the result.
    eg. searches a string for the character "e":

    /e/.test("The best things in life are free!");
// returns "true"

    -exec: searches a string for a specified pattern, and returns the found text as an object.
    eg. searches a string for the character "e":

    var obj = /e/.exec("Best things are free!");
document.getElementById("demo").innerHTML = obj[0]+" position:"+obj.index +" text:"+obj.input;
// obj[0] = obj = e;
// obj.index = first obj (e) occurs;
// obj.input = ("text balabala");

Scope

Each function creates a new scope. Scope determines the accessibility (visibility) of variables.

  1. Local scope: Variables declared within a function.
    Local variables have Function scope: They can only be accessed from within the function.
    //* Local variables are created when a function starts, and deleted when the function is completed.

  2. Global scope: Variables declared outside a function.
    Global variables have Global scope: All scripts and functions on a web page can access it.
    //* In JavaScript, objects and functions are also variables.

    • Window Object: In HTML, the global scope is the window object. All global variables belong to the window object.
      //* Do NOT create global variables unless you intend to. Global variables (or functions) can overwrite window variables (or functions).
      var carname = "Volvo";

// code here can use window.carname
    • Automatically Global: If you assign a value to a variable that has not been declared, it will automatically become a GLOBAL variable.
      myFunction();

// carname becomes a global variable

function myFunction() {
carname = "Volvo";
}
    • Strict Mode: Global variables are not created automatically in "Strict Mode".

"use strict"; defines to the beginning of a script / function that JS code should be executed in "strict mode".

Hoisting

Hoisting is JavaScript's default behavior of moving declarations to the top.
//* JavaScript only hoists declarations (var x;), not initializations (var x = 10;).
//* Variables and constants declared with let or const are not hoisted!
//* They provides Block Scope: variables declared inside a block{} can NOT be accessed from outside the block.

  1. let:
    • Declare:
    { var x = 2; }
// (variables declared with var can NOT have Block Scope)
// x can be used here

{ let x = 2; }
// (variables declared with let can have Block Scope)
// x can NOT be used here
    • Loop Scope:
    var i = 5;
for (let i = 0; i < 10; i++) { /* some statements */ }
// Here i is 5
    • Redeclaring: allowed in different block.
    var x = 2;       // Allowed
let x = 3;       // Not allowed
{
    let x = 4;   // Allowed
    var x = 5;   // Not allowed
}
    • Hoisting:
    //you can NOT use carname here
let carname;

//you CAN use carname here
var carname;
  2. const:
    • Declare:
      //* It does NOT define a constant value. It defines a consant reference to a value.
    const PI;           // must be assigned a value when they are declared
PI = 3.14159;       // give an error

cost PI = 3.14159;  // correct
    • Reassign:
      • Primitive Value: can NOT change.
        const PI = 3.14159; // can NOT be reassign

PI = 3.14;          // give an error
PI = PI + 10;       // give an error
      • Object properties: can change.
        const car = { type:"Fiat", Model:"500", color:"white"};

car.color = "red";      // can change a property
car.owner = "Johnson";  // can add a property
      • Array elements: can change.
        const car = ["Saab", "Volvo", "BMW"];

cars[0] = "Toyota"; // can change an element
car.push("Audi");   // can add an element
    • Redeclaring: allowed in different block.
    const x = 2;    // allowed   
const x = 3;    // not allowed
x = 3;          // not allowed
var x = 3;      // not allowed
let x = 3;      // not allowed
{
    const x = 3; // allowed
}
    • Hoisting: NOT hoisted to the top, can NOT be used before it is declared.

  5. In an event, this refers to the element handler that received the event.

     // refers to the HTML element that receive the event (button)
  6. Methods like call(), and apply() can refer this to any object.

  3. Static Methods: defined on the class itself, and not on the prototype. Means can NOT call it on object(mycar), but on class(Car):
    class Car {
    constructor(brand) {
        this.carname = brand;
    }
    static hello(x) {
        return "Hello!" + x.carname;
    }
}

mycar = new Car("Ford");

document.getElementById("demo").innerHTML = mycar.hello(mycar);
// cause an error

document.getElementById("demo").innerHTML = Car.hello(mycar);
// correct
  4. Inheritance: use keyword extend. A class inherits all the methods from another class.
    class Car {
    constructor(brand) {
        this.carname = brand;
    }
    present() {
        return 'I have a ' + this.carname;
    }
}

class Model extends Car {
    constructor(brand, mod) {
        super(brand);
        this.model = mod;
    }
    show() {
        return this.present() + ', it is a ' + this.model;
    }
}

mycar = new Model("Ford", "Mustang");
document.getElementById("demo").innerHTML = mycar.show();
// returns "I have a Ford, it is a Mustang"
  5. Getters and Setters: do sth with the value before returning them or before set them.
    //* You can use underscore character to seperate the getter/setter from actual property.
    class Car {
    constructor(brand) {
        this._carname = brand;
    }
    get cnam() {
        return this._carname;
    }
    set cnam(x) {
        this._carname = x;
    }
}

mycar = new Car("Ford");
document.getElementById("demo").innerHTML = mycar.cnam;
// do NOT use parentheses when you want to get the property value.

mycar.carname = "Volvo";
document.getElementById("demo").innerHTML = mycar.carname;
// without parentheses.
  6. Hoisting: class declarations are NOT hoisting. Means that must declare class before use it.

Mistake

Common Mistake:

  1. Accidentlly using the Assignment Operator.
    var x = 0;
if (x == 10) // returns false, because x is not equal to 10
if (x = 10)  // returns true, because 10 is true
if (x = 0)   // returns false, because 0 is false
  2. Expecting Loose Comparison:
    var x = 10;
switch(x) {
    case 10: alert("Hello");   // will display an alert
}
switch(x) {
    case "10": alert("Hello"); // will NOT display an alert
}
  3. Confusing Addition & Concatenation:
    • Addition: adding numbers.
    • Concatenation: adding strings.
    var x = 10 + 5;   // x is 15
var x = 10 + "5"; // x is "105"
  4. Misunderstanding Floats: All numbers are stored as 64-bits loating point numbers.
    x = 0.1;
y = 0.2;
var z = x + y;                     // z will NOT be 0.3
var z = (x * 10 + y * 10) / 10;    // z will be 0.3
  5. Reserved Words: Details

Performance

speed up JS code.

  1. reduce activity in loop: each statement in a loop is executed for each iteration of the loop.
    var i;
for (i = 0; i < arr.length; i++) {
// bad

var i;
var l = arr.length;
for (i = 0; i < l; i++) {
//better
  2. reduce DOM access: accessing HTML DOM is very slow. Access it once, and use it as a loal variable.
    var obj = document.getElementById("demo");
obj.innerHTML = "Hello";
  3. Avoid unecessary variables": do NOT create new variables if you don't plan to save values.
    var fullName = firstName + " " + lastName;
document.getElementById("demo").innerHTML = fullName;
// bad

document.getElementById("demo").innerHTML = firstName + " " + lastName;
// better
  4. Delay JS loading: putting scripts at the bottom of the page body, lets the browser load the page first.
  5. Avoid new:
    {} instead of new Object()
    "" instead of new String()
    12345 instead of new Number()
    true/false instead of new Boolean()
    [] instead of new Object()
    /()/ instead of new RegExp()
    function() {} instead of new function()
